# Шаблонизатор документов ПДн

Веб-приложение для одной организации: заполните реквизиты и повторяющиеся
блоки данных один раз — получите полный пакет из 7 документов по 152-ФЗ
(Положение, Политика, Акт уничтожения, два Журнала, Согласие, Акт оценки
вреда) в формате .docx (и .pdf — там, где доступен LibreOffice, см. ниже) с
автоматически подставленными данными.

## Стек

- **Next.js 15 (App Router) + TypeScript** — единое full-stack приложение,
  рассчитанное на деплой на Vercel (бесплатный тариф) без своего сервера.
- **Prisma + Postgres** — хранение реквизитов организации и четырёх коллекций
  (цели обработки, перечень ПДн, перечень ИСПДн, оценка вреда). В проде —
  Vercel Postgres/Neon (см. «Деплой на Vercel» ниже); локально можно
  использовать ту же облачную БД или любой другой Postgres.
- **docxtemplater + pizzip** — рендеринг .docx по тегам, включая циклы по
  строкам таблиц и повторяющиеся блоки (свободный MIT/GPL-core, без платных
  модулей).
- **LibreOffice (headless), опционально** — конвертация .docx → .pdf через
  `soffice --convert-to pdf`. Приложение само определяет, доступен ли
  `soffice` в текущем окружении (`src/lib/pdfSupport.ts`), и просто прячет
  кнопку «.pdf», если нет — на Vercel serverless бинарник поставить нельзя,
  поэтому там штатно работает только .docx.

## Быстрый старт (локально)

```bash
npm install
# создать .env с DATABASE_URL — см. .env.example (нужен Postgres, см. ниже)
npx prisma migrate dev   # применяет prisma/migrations/
npm run dev              # http://localhost:3000/form
```

Нужен настоящий Postgres — SQLite/файловая БД больше не используется (не
переживает serverless-деплой на Vercel). Проще всего взять бесплатный: зайти
на [neon.com](https://neon.com) (или создать Vercel Postgres из панели Vercel
— это тот же Neon под капотом), создать проект, скопировать connection string
в `DATABASE_URL`. Локальная разработка и продакшн могут использовать один и
тот же облачный Postgres — это единственная база в проекте, ничего не течёт.

LibreOffice — опционально, только если нужна кнопка «.pdf» локально:
`winget install TheDocumentFoundation.LibreOffice` (Windows). Путь к бинарнику
можно переопределить переменной окружения `LIBREOFFICE_BINARY_PATH`.

## Деплой на Vercel

1. Запушить репозиторий на GitHub, импортировать в Vercel — Next.js
   определяется автоматически, доп. настройка не нужна.
2. В настройках проекта на Vercel добавить `DATABASE_URL` (Environment
   Variables) — connection string от Vercel Postgres/Neon.
3. Vercel сам подхватит скрипт `vercel-build` (`prisma migrate deploy && next
   build`) вместо обычного `next build` — миграции применяются автоматически
   при каждом деплое, руками ничего накатывать не нужно.
4. Опционально — задать `APP_PASSWORD` и `SESSION_SECRET`, если нужен
   пароль на вход (см. ниже).
5. Deploy. PDF-кнопки в интерфейсе не будет (на Vercel нет LibreOffice) —
   пользователь скачивает .docx и при необходимости сохраняет как PDF из
   Word/Google Docs сам, в 1 клик.

Всё бесплатно на Hobby-тарифах Vercel + Neon при небольшой нагрузке (один
арендатор, нечастая генерация документов) — платного сервера не требуется.

## Пароль на вход (опционально)

По умолчанию приложение открыто без входа — подходит для запуска в частной
сети. Чтобы включить простой pin/пароль-гейт, задайте в `.env`:

```
APP_PASSWORD="ваш-пароль"
SESSION_SECRET="случайная строка не короче 32 символов"
```

Сессия — подписанная cookie (`iron-session`), без реальной многопользовательской
системы аутентификации — это тонкий, снимаемый слой, а не полноценный auth.

## Docker (альтернатива Vercel — если всё же нужен PDF на сервере)

```bash
docker build -t pdn-doc-templater .
docker run -p 3000:3000 -e DATABASE_URL="postgresql://..." pdn-doc-templater
```

Образ включает `libreoffice-writer` (не весь офисный пакет) и `fonts-liberation`
— метрически совместимые замены Times New Roman/Arial с полной поддержкой
кириллицы (в оригинальных .docx использован Times New Roman, которого нет на
Linux) — то есть в этом варианте кнопка «.pdf» работает (LibreOffice есть).
БД по-прежнему внешний Postgres (тот же Neon подойдёт) — своей базы в образе
нет. Миграции применяются автоматически при старте контейнера
(`prisma migrate deploy`).

Держать этот путь имеет смысл, только если PDF из коробки важнее, чем
"нулевая инфраструктура" — сам контейнер уже не бесплатный/serverless, нужен
хостинг с постоянным процессом (Fly.io, Render и т.п.), у большинства из них
свободный тариф либо ограничен, либо "засыпает" при простое.

**Важно:** Dockerfile не был протестирован реальной сборкой — на этой машине
нет Docker. Логика (multi-stage build, LibreOffice + шрифты) стандартна, но
перед использованием стоит один раз собрать и прогнать образ локально.

## Как это устроено

Исходные документы лежат в `exampleDocs/` (нетронутые, это юридический
first-source). Настоящие merge-шаблоны — в `templates/masters/` — они
сгенерированы из `exampleDocs/` скриптами в `scripts/reauthor/`.

Авторы исходных .docx помечали места для заполнения жёлтой заливкой текста
(`w:highlight val="yellow"`) — это надёжный маркер, который позволил
скриптам находить и заменять плейсхолдеры программно, без риска зацепить
похожий текст в другом месте (например, «ИНН» как подпись поля и «ИНН» как
значение в одной таблице). Скрипты re-authoring — не часть рантайма
приложения, это одноразовый (per-document) build-инструмент:

```bash
npx tsx scripts/reauthor/build-polozhenie.ts    # exampleDocs -> templates/masters/polozhenie.docx
npx tsx scripts/reauthor/build-politika.ts      # exampleDocs -> templates/masters/politika.docx
npx tsx scripts/reauthor/build-simple-docs.ts   # Акт уничтожения + оба Журнала + Согласие (только реквизиты, без циклов)
npx tsx scripts/reauthor/build-akt-vreda.ts     # Акт оценки вреда (два независимых цикла по harmCategories)
npx tsx scripts/lint-templates.ts               # рендерит все шаблоны на фикстурах, падает при неразрешённых тегах
```

Каждый шаблон использует три вида тегов:
- **Скалярные** — `{org_short}`, `{org_inn}` и т.д., подставляются столько
  раз, сколько встречаются в документе.
- **Циклы по строкам таблицы** (Приложение №2/№3 — «Перечень ПДн», «Перечень
  ИСПДн»; сводная таблица в Акте оценки вреда) — `{#pdItems}...{/pdItems}` в
  первой/последней ячейке строки, docxtemplater размножает строку на каждый
  элемент массива.
- **Циклы по блоку** (Приложение №1 Положения — «Цели обработки»; чек-лист
  Акта оценки вреда) — весь блок (заголовок + таблица) обёрнут в
  `{#purposes}...{/purposes}` / `{#harmCategories}...{/harmCategories}` и
  повторяется целиком — оригинальный, более читаемый вид сохранён вместо
  сжатия в одну широкую таблицу.

Акт оценки вреда (Приказ Роскомнадзора №178) устроен сложнее остальных: у него
**два независимых цикла** по одной и той же коллекции `harmCategories` — сводная
таблица (категория → степень вреда) и, отдельно, детальный чек-лист из ~13
фиксированных законом параметров на категорию (осуществляется/не
осуществляется). Формулировки параметров — фиксированный текст закона,
хранятся только в `src/lib/harmParams.ts` (единый источник правды для формы,
валидации и генерации) и в самом шаблоне; в БД хранятся только да/нет-ответы
(`HarmCategory.answers`, JSON). Степень вреда (высокая/средняя/низкая) нигде не
хранится — она **вычисляется** из ответов при каждой генерации
(`computeHarmDegree`), поэтому не может разойтись с чек-листом.

## Структура

```
exampleDocs/            оригинальные 7 документов — не редактируются
templates/masters/      сгенерированные merge-шаблоны (используются в рантайме)
scripts/reauthor/       build-скрипты exampleDocs -> templates/masters
scripts/lint-templates.ts   рендер-валидация на фикстурах
prisma/schema.prisma    Organization + Purpose + PdItem + Ispdn + HarmCategory (Postgres)
src/lib/harmParams.ts   13 фиксированных параметров оценки вреда + вычисление степени
src/lib/pdfSupport.ts   определяет, доступен ли soffice в текущем окружении (кэшируется)
src/lib/                организация данных, генерация docx/pdf, схемы валидации
src/components/         форма (Реквизиты + 4 повторяющихся блока) и панель скачивания
src/app/form/           страница ввода данных
src/app/api/            organization CRUD + generate/[doc], generate/package
src/app/login/          страница входа (активна только если задан APP_PASSWORD)
src/middleware.ts       опциональный auth-гейт (redirect на /login, 401 для API)
Dockerfile              multi-stage сборка + LibreOffice в рантайм-образе
```

| Акт оценки вреда | категории субъектов × 13 да/нет вопросов о вреде | ✅ |
